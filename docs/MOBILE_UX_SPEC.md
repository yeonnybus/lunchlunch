# Mobile UX Spec

## 목표
- 360px~430px 뷰포트에서 로그인/크롤링/추천 플로우가 한 화면 내 자연스럽게 동작
- 터치 중심 인터랙션(버튼 높이, 입력 간격, 링크 클릭 영역) 확보

## 주요 화면 구성
- 상단: 서비스 소개
- 본문 1: 로그인 카드(Google 로그인/로그아웃)
- 본문 2: 크롤링 카드(지역/쿼리/토큰)
- 본문 3: 추천 카드(취향 메모 + 실행 버튼)
- 하단: 추천 결과 카드(피치 문장 + 메뉴 배지 + 식당 목록)

## 반응형 규칙
- 모바일 기본 1열, `lg`에서 2열 배치
- 입력/버튼은 `w-full` 유지
- 추천 결과 카드 링크는 카드 전체가 클릭 영역

## 검증 체크리스트
- [ ] Google 로그인 후 리디렉션 복귀
- [ ] 새로고침 후 로그인 유지
- [ ] 추천 실행 시 로딩 상태 명확 표시
- [ ] 도보 버킷 배지가 작은 화면에서 줄바꿈 없이 표기
